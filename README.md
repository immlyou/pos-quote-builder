# POS Quote Builder

Internal web app for quoting Partner Tech Windows POS / Box PC products.

## Quick Start

```bash
cp .env.example .env.local
# edit .env.local and set APP_PASSWORD
npm install
npm run dev
```

Open http://localhost:3000 — you'll be redirected to `/login`. Enter the password from `.env.local`.

## Environment Variables

| Variable | Description | Default |
|---|---|---|
| `APP_PASSWORD` | Shared login password | `changeme` |

## Updating the Catalog

1. Drop a new `data/source.xlsx` into the project root.
2. Run `npm run data:import` — this regenerates `src/data/catalog.json`.
3. Commit both files and redeploy.

```bash
npm run data:import
git add src/data/catalog.json data/source.xlsx
git commit -m "chore: update catalog"
git push
```

## Vercel Deploy

1. Push to GitHub.
2. Import project in Vercel — framework auto-detected as Next.js.
3. Add environment variable `APP_PASSWORD` in Vercel project settings.
4. Deploy. Every `git push` triggers a new deploy automatically.

## Features

- **Model Selection** — filter by series/terminal/platform, pick one base model
- **Upgrades** — toggle upgrade options and add-on modules
- **Peripherals** — auto-filtered by selected model's series, quantity + MOQ tier pricing
- **Licenses** — per-OS license with platform suggestion hint
- **Others** — M10, KDS, Stands, IO Box, Payment Brackets, IoT via tabbed view
- **Summary panel** — live totals, customer/quote metadata
- **Export** — PDF (A4 professional layout) and Excel (.xlsx) with company logo
- **Company Profiles** — CRUD profiles with logo; stored in localStorage
- **Draft** — save/load quote state to localStorage
- **i18n** — zh-TW / EN toggle (UI chrome only; product descriptions unchanged)
- **Auth gate** — shared password via cookie; lightweight, internal use only
